## What Is ContractIQ?

AI-powered procurement intelligence for the City of Richmond. Staff upload contract PDFs, the system extracts structured data via OCR + AI, validates for consistency, routes to the correct department, and surfaces expiration risks on a live dashboard — using real City of Richmond contract data from the Socrata Open Data Portal.

**Not a legal compliance tool.** ContractIQ is a decision-support system. All AI-assisted extractions require human review before action.

---

## Architecture

```
Staff uploads PDF/image → FastAPI backend → Azure Blob Storage
                                                ↓
                                    Azure Document Intelligence (OCR)
                                                ↓
                                    Azure OpenAI GPT-4 mini (classify + extract)
                                                ↓
                                    Validation Engine (16 rules + AI)
                                                ↓
                                    PostgreSQL (structured data)
                                                ↓
                         Next.js 16 Dashboard ← REST API (polling)
```

| Layer | Technology |
|---|---|
| Backend | FastAPI, SQLAlchemy 2.0 async, Python 3.14 |
| Frontend | Next.js 16, React 19, TanStack Query, Tailwind CSS 4 |
| OCR | Azure Document Intelligence (prebuilt-read) |
| AI | Azure OpenAI GPT-4 mini (JSON Schema structured output) |
| Storage | Azure Blob Storage + Azure PostgreSQL Flexible Server |
| Data | Socrata CSV (~1,365 real City contracts) + 10 pre-staged PDFs |
| Deployment | Azure Container Apps (Consumption plan) |

## Feature Summary

### Data Processing Pipeline

**6-step async pipeline** runs as a background task after each upload:

1. **Upload** — Stream to Azure Blob Storage with SHA-256 dedup
2. **OCR** — Azure Document Intelligence with automatic chunking for large PDFs (>3.5MB split by page range, 100-page budget)
3. **Classify** — GPT-4 mini categorizes into 8 document types (contract, RFP, RFQ, PO, invoice, amendment, cooperative, other)
4. **Extract** — GPT-4 mini extracts 35 structured fields with per-field confidence scores
5. **Validate** — 16 rule-based checks + AI cross-field consistency
6. **Route** — Auto-assign primary department from 14-code taxonomy + log routing decision

**Resilience:** 3 retries with exponential backoff (2s/4s/8s), 30s AI timeout, 120s OCR timeout, concurrency semaphore (max 3 concurrent), stale document recovery on server restart.

### Extracted Intelligence (35 fields per document)

| Category | Fields |
|---|---|
| **Core** | Title, document number, vendor name, issuing department, total amount, currency, document date, scope summary |
| **Dates** | Effective date, expiration date, expiration date source (exact quote) |
| **Contract Terms** | Contract type, payment terms, renewal clause, insurance required, bond required |
| **Department Routing** | Primary department, department tags (1-3), department confidence |
| **Compliance** | MBE/WBE required, MBE/WBE details, federal funding, compliance flags (MBE_WBE, DAVIS_BACON, ADA, DRUG_FREE_WORKPLACE, OSHA, VDOT_STANDARDS, ENVIRONMENTAL, EEO) |
| **Insurance & Bonding** | General liability min, auto liability min, professional liability min, workers' comp required, performance bond amount, payment bond amount, liquidated damages rate |
| **Procurement** | Procurement method (competitive bid, cooperative, sole source, RFP, emergency), cooperative contract reference, prequalification required |
| **Confidence** | Overall extraction confidence, per-field confidence scores (9 critical fields) |

### Validation Engine (16 rules + AI)

| Rule | Severity | What It Catches |
|---|---|---|
| DATE_LOGIC | Error | Expiration before effective date |
| CONTRACT_EXPIRED | Error | Already-expired contracts |
| CONTRACT_EXPIRING_30 | Error | Expires within 30 days |
| MISSING_EXPIRATION | Error | Contract/PO with no expiration date |
| CONTRACT_EXPIRING_90 | Warning | Expires within 90 days |
| HIGH_VALUE_NO_BOND | Warning | >$100K without performance bond |
| MISSING_AMOUNT | Warning | Contract type but no total amount |
| MISSING_VENDOR | Warning | No vendor name extracted |
| AMOUNT_RANGE | Warning | >$10M (likely OCR error) |
| LOW_OCR_CONFIDENCE | Warning | OCR confidence below 85% |
| LOW_CLASSIFICATION | Warning | Document type confidence below 75% |
| LOW_EXPIRATION_CONFIDENCE | Warning | Expiration date confidence below threshold |
| LOW_FIELD_CONFIDENCE | Warning | Critical field confidence below threshold |
| DATE_RANGE | Warning | Dates outside 2000-2035 (catches AI hallucinations) |
| COMPLIANCE_MISSING | Warning | >$100K without MBE/WBE clause |
| BOND_AMOUNT_MISMATCH | Warning | Construction contract where bond != contract amount |
| MISSING_INSURANCE | Info | >$50K without insurance requirement |
| AI_CONSISTENCY | Warning | Cross-field logical inconsistencies (via GPT-4 mini) |

### Data Ingestion

**Socrata CSV Import:**
- Downloads ~1,365 real City of Richmond contracts from `data.richmondgov.com`
- Flexible column name matching (handles `contract_number`, `Contract Number`, `contract_#`, etc.)
- Date parsing handles `MM/DD/YYYY HH:MM:SS AM` format
- Deduplication via document number or vendor+dates composite key
- Batch inserts (200 records per flush)
- Auto-populates department routing, procurement method, and compliance flags

**Uploaded PDFs:**
- 10 pre-staged real Richmond contract PDFs
- Full AI pipeline processing (OCR → classify → extract → validate)
- Demo cache mode for offline presentations

### Approval Workflow

```
uploaded → processing → extracted → analyst_review → pending_approval → approved / rejected
```

| Role | Capabilities |
|---|---|
| **Analyst** | Upload documents, review extracted fields, resolve warnings, submit for approval |
| **Supervisor** | Approve/reject with comments, override fields, reprocess documents |

Separation of duties enforced: analysts cannot approve their own reviews.

### Security & Upload Validation

- Magic bytes verification (rejects renamed executables)
- File size limit (50MB) with streaming upload (no full-file memory load)
- Allowed MIME types: PDF, PNG, JPEG, TIFF
- SHA-256 duplicate detection
- Upload rate limiting (5 per 5 minutes per user)
- CORS restricted to configured origins
- All UUIDs (no sequential IDs)
- Immutable activity log audit trail

## Known Limitations (v1)

- **Authentication:** Client-side localStorage role selector (no real auth)
- **Chat:** Keyword search + LLM synthesis (not true vector-based RAG)
- **AI Opportunity Feed:** Hardcoded demo items (not real-time ML analysis)
- **Savings metrics:** Demo placeholder ($42K)
- **Polling:** 30-second interval on list, 5-second on detail during processing (no WebSocket)
- **Uploaded PDFs:** 4 of 10 in error state (Azure DI credentials needed for full processing)
